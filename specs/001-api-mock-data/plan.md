# Implementation Plan: API Layer with Mock Data

**Branch**: `001-api-mock-data` | **Date**: 2026-02-11 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-api-mock-data/spec.md`

## Summary

Migrate all 8 pages from static mock data imports (`src/data/mockData.ts`) to Next.js API Route Handlers. Mock data arrays are preserved as the data source within the route handlers, served with server-side filtering, sorting, and search via query parameters. Zod validates all incoming query parameters with Arabic error messages. Pages call endpoints via `fetch()` instead of importing data directly.

## Technical Context

**Language/Version**: TypeScript 5.x (strict mode)
**Primary Dependencies**: Next.js 16.x (App Router), React 19.x, Zod 4.x
**Storage**: In-memory mock data arrays (no database this phase)
**Testing**: `npm run lint` + `npm run build` per-task gate; Playwright available for E2E
**Target Platform**: Web (Node.js server + browser client)
**Project Type**: Single Next.js project
**Performance Goals**: Sub-1-second filter/sort response, instant search results
**Constraints**: Mock data only (8 prompts, 8 categories, 3 reviews, 3 testimonials). No pagination. No database.
**Scale/Scope**: 8 pages migrated, 7 API route handlers created, Zod schemas for all endpoints
